# Start a New Project with Context Spine

```txt
Start this new project with Context Spine using https://github.com/BerniceHole/context-spine.

Use Context Spine as the working protocol and small operating layer for canonical state, task contracts, authority, execution evidence, continuity, and agent behavior. Confirm the designated project location, project identity, and current write authority.

When the human explicitly designates an empty non-Git directory as a new repository-backed project, local bootstrap authority includes:
- creating the designated root if needed;
- initializing Git unless explicitly forbidden;
- creating a minimal .gitignore for incidental OS or editor metadata encountered during bootstrap;
- creating only the Spine Four: AGENTS.md and ai-state/CURRENT_STATE.md, DECISIONS.md, HANDOFF.md;
- recording the revision as unborn HEAD until a first commit exists;
- making the first local commit only when the chosen integration outcome authorizes it.

Do not assume a branch name. Follow explicit project policy or the configured Git default.

Classify unexpected contents before bootstrap.

Material mismatches that require a safe stop:
- existing Git history;
- secrets or credentials;
- conflicting project files;
- unrelated source;
- ambiguous project root;
- contradictory project identity.

Incidental mismatches that may be ignored or minimally excluded:
- .DS_Store;
- harmless ignored editor caches;
- equivalent OS metadata;
- harmless generated files that can be ignored without changing project identity.

Only material conflicts require a safe stop. Do not authorize remote push, deployment, publication, dependency installation, or product or architecture invention through this bootstrap rule.

Do not design the product structure, docs tree, roadmap, architecture, design system, brand system, information architecture, or domain model unless explicitly asked. Keep placeholders short. Do not add optional docs unless a real repeated failure justifies them. Ask before making product or architecture assumptions.
```
