# Quality Bar

Use this optional file only when quality criteria are repeatedly forgotten or applied inconsistently.

## General criteria

- The output satisfies the accepted work order.
- Scope is controlled; non-goals were not touched.
- Edge states are considered where relevant.
- Accessibility is checked in actual use, not only in markup.
- Public claims are supported by evidence.
- Validation evidence is recorded as `Passed`, `Failed`, or `Not run`.
- State documents remain accurate after the change.

Add project-specific criteria only when they prevent repeated quality drift.
