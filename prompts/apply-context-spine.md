Apply Context Spine to this existing project using https://github.com/BerniceHole/context-spine.

Context Spine is the working protocol and small operating layer for project state, decisions, handoff, validation, and agent behavior. It does not replace the target project's product structure, documentation architecture, design system, roadmap, information architecture, brand system, or domain model.

Inspect the target project first. Install or update only the Spine Four by default: AGENTS.md, ai-state/CURRENT_STATE.md, ai-state/DECISIONS.md, and ai-state/HANDOFF.md. Do not implement features. Do not copy secrets or private data. Use Unknown where evidence is missing. Report files changed and validation as Passed, Failed, or Not run.
